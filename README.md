# GPT分析器

本工具用于分析 GPT 账号池截图，并保留原来的成本计算功能。

## 账号池分析

1. 打开 `GPT分析器.app`。
2. 复制 `PLUS共享号池` 或 `K12共享号池` 看板截图。
3. 回到应用按 `Command+V`。
4. 应用会自动识别截图标题，保存到对应分组历史。
5. 在“账号池分析”里查看上方 PLUS/K12 概览，以及下方两个独立历史表里的账号总数、5h/7d 可调度剩余、并发可用、状态分布和趋势变化。

账号池历史按分组独立保存：PLUS 只和 PLUS 的上一次对比，K12 只和 K12 的上一次对比。历史记录不限制条数。

## 成本计算

原来的成本计算功能仍保留：

1. 输入成本，例如 `200`。
2. 复制基准截图，点击“粘贴为基准截图”。
3. 后续复制最新截图，点击“粘贴为最新截图”。
4. 查看各号余额对比、当前收益和回本进度。

## 隐私和 OCR

- OCR 使用 macOS 系统自带 Vision 框架，本地识别，不上传图片。
- 图片只用于当次 OCR，不复制、不保存图片路径。
- 应用只保留识别出来的结构化数字、分组和时间。

## 重新打包

在项目目录运行：

```bash
./build.sh
```

生成文件在：

```text
dist/GPT分析器.dmg
dist/GPT分析器.zip
```

构建脚本会下载并校验 Sparkle 2.9.4，生成同时支持 Apple 芯片和 Intel Mac 的通用应用。

## 自动更新与发布

- 应用通过 Sparkle 每小时自动检查一次更新。
- 也可以在应用菜单中点击“检查更新…”。
- 更新清单和安装包来自 GitHub Releases，更新文件使用 EdDSA 签名验证。
- 没有 Apple Developer ID，首次从 GitHub 安装时需要在 macOS“隐私与安全性”中手动允许打开。

发布新版本前先更新 `RELEASE_NOTES.md`，然后运行：

```bash
./release.sh 1.0.1
```

脚本会递增内部构建号，在本机完成构建和 Sparkle 签名，然后提交版本、推送标签并创建 GitHub Release。Sparkle 私钥只保存在本机钥匙串，不会上传到 GitHub。

GitHub Actions 只负责对 `main` 分支进行独立的通用构建校验，不接触更新私钥。
